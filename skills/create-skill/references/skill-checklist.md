# Skill 质量检查清单

生成 skill 后，逐项对照此清单进行质量检查。

## 结构完整性

- [ ] 存在 `skills/<skill-name>/SKILL.md` 文件
- [ ] 存在 `skills/<skill-name>/README.md` 文件
- [ ] SKILL.md 包含有效的 YAML frontmatter
- [ ] frontmatter 包含 `name` 字段
- [ ] frontmatter 包含 `description` 字段
- [ ] 正文包含 `## 概述` 部分
- [ ] 正文包含 `## 工作流程` 部分
- [ ] 如有参考文件，位于 `references/` 目录下

## Frontmatter 质量

- [ ] `name` 使用小写英文+连字符格式
- [ ] `name` 与目录名一致
- [ ] `description` 使用 `>-` 多行折叠格式
- [ ] `description` 第一句话概述功能
- [ ] `description` 包含中文触发词
- [ ] `description` 包含英文触发词
- [ ] `description` 说明激活条件

## 工作流程质量

- [ ] 工作流程分阶段，结构清晰
- [ ] 每个阶段有明确的步骤编号
- [ ] 需要用户确认的节点已显式标注
- [ ] 关键阶段有输出格式模板
- [ ] 输出格式用代码块包裹
- [ ] 阶段间的推进条件明确

## 规则质量

- [ ] 规则使用列表形式
- [ ] 包含"必须做"类规则
- [ ] 包含"绝对不能做"类约束
- [ ] 每条规则独立、可执行
- [ ] 规定了与用户的沟通语言策略

## 触发词质量

- [ ] 触发词覆盖用户常见的口语化表达
- [ ] 触发词覆盖正式表达
- [ ] 中英文均有覆盖
- [ ] 与已有 skill 的触发词无冲突
- [ ] 触发词足够具体，不会误触发

## 参考文件质量（如适用）

- [ ] 文件名使用小写英文+连字符
- [ ] 使用 Markdown 格式
- [ ] 用二级标题按类别分节
- [ ] 内容使用列表或表格组织
- [ ] 无大段叙述性文字

## Skill 独立 README

- [ ] README.md 位于 skill 目录下
- [ ] 包含 skill 名称作为一级标题
- [ ] 包含一句话功能描述
- [ ] 包含功能特性列表
- [ ] 包含执行流程（按阶段描述）
- [ ] 包含安装命令
- [ ] 包含使用方式（触发词列表）
- [ ] 包含文件结构树

## 项目根目录 README 更新

- [ ] Skills 列表表格中已新增该 skill 的行
- [ ] 已新增该 skill 的二级章节（核心流程 + 功能特性）
- [ ] 安装章节中已补充该 skill 的单独安装命令
- [ ] 使用方式章节中已列出该 skill 的触发词
- [ ] 项目结构目录树中已补充该 skill 的文件结构

## 可用性

- [ ] 概述清晰定义了 Agent 角色
- [ ] 步骤描述可操作，无模糊指令
- [ ] 对大型任务有拆分建议
- [ ] 异常情况有处理指引（如用户描述模糊时怎么办）
